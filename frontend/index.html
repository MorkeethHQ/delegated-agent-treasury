How it works

Five layers. One product.

1. Treasury Primitive

Lido wstETH on Base

Human deposits wstETH. Lido staking rewards cause the exchange rate to rise, generating yield at ~3.5% APY. The contract records the rate at deposit time. Available yield = current value minus principal value. The agent can spend up to that delta. Principal is structurally locked at the EVM level.

2. Control Layer

Policy Engine

Every action passes through a policy engine: per-tx caps, daily aggregate limits, recipient whitelist, approval thresholds. Small spends auto-execute. Large spends require human approval. Denied actions are blocked and logged.

3. Trust Layer

ERC-8004 + MetaMask Delegations

On-chain agent identity gates trust decisions. MetaMask Delegation caveats (ERC-7710) enforce the same constraints onchain as the policy engine does offchain. Even if the offchain layer is bypassed, onchain caveats protect the treasury.

4. Execution Layer

Uniswap + MoonPay + x402

Agents deploy yield through policy-gated Uniswap swaps on Base. MoonPay CLI provides an alternative multi-chain execution backend. x402 lets other agents pay USDC to access this treasury as a service.

5. Portability

Base + Celo

Base is the primary home (ETH staking yield). Celo proves portability (stataUSDC lending yield from Aave). Different yield source, different risk profile, identical spending constraint.

Run it yourself

Clone, configure, and give your agent a yield-funded budget in 5 minutes.

For Humans Deploy a treasury

git clone https://github.com/MorkeethHQ/delegated-agent-treasury
cd delegated-agent-treasury
npm install && npm run build

# Configure your keys
cp .env.example .env
# Set AGENT_PRIVATE_KEY, OWNER_PRIVATE_KEY,
# TREASURY_ADDRESS, WSTETH_ADDRESS

# Start the API
node --env-file=.env dist/apps/api/src/server.js

# Check your treasury
curl http://localhost:3001/treasury

For Agents Connect and spend

# Discover capabilities
curl http://localhost:3001/onboarding/status

# Check available yield
curl http://localhost:3001/treasury

# Submit a spending plan
curl -X POST http://localhost:3001/plans/evaluate \
  -H 'content-type: application/json' \
  -d '{
    "planId": "plan-1",
    "agentId": "my-agent",
    "type": "transfer",
    "amount": 0.005,
    "destination": "0x...",
    "reason": "Fund operations"
  }'
