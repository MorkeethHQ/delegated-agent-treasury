5-Layer Architecture

From on-chain primitives to cross-chain portability

Layer 1
Treasury Primitive
Yield-bearing deposit with structural principal lock. Agent spends only accrued yield, enforced at the Solidity level.

Layer 2
Control Layer
Policy engine with per-action caps, daily limits, recipient whitelists. Multi-agent roles: proposer, executor, auditor with freeze circuit breaker.

Trust Layer

Execution Layer

How It Works

The autonomous yield spending loop

Human deposits wstETH
Principal locked in treasury contract. Exchange rate recorded at deposit time.
Yield accrues via staking
wstETH exchange rate rises. Delta between current and deposit rate = available yield.
Agent submits spending plan
Policy engine evaluates: caps, whitelists, daily limits. Auto-approve, escalate, or deny.
spendYield() executes on-chain
Only available yield is transferable. Contract reverts if amount exceeds yield.
Audit logged, loop repeats
Append-only JSONL log records everything. Agent waits for more yield to accrue.
