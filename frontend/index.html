Two interfaces

For humans. For agents.

Humans set the rules. Agents execute within them. Same treasury, different entry points.

For Humans

You control the treasury

  • Deposit wstETH — lock principal, yield becomes the budget
  • Set per-transaction spending caps
  • Whitelist approved recipient addresses
  • Review and approve large spending plans
  • Freeze any agent instantly
  • Full audit trail of every action
  • Principal always withdrawable. You stay in control.
For Agents

Autonomous operator

  • Query available yield — GET /treasury
  • Submit spending plans — POST /plans/evaluate
  • Auto-approved if within caps, escalated if over
  • Deploy yield via Uniswap swaps
  • Self-discovery — GET /onboarding/status
  • 24 MCP tools for native agent integration
  • Budget regenerates from staking yield. Never runs out.
Architecture

Five layers. One product.

From the yield source to cross-chain portability. Every layer serves bounded authority.

Treasury Primitive

Lido wstETH

Human deposits wstETH. Staking rewards generate yield at ~3.5% APY. Agent spends up to the yield delta. Principal locked at the EVM level. $10K deposit = ~$350/yr autonomous budget.

Control Layer

Policy Engine

Every action evaluated: per-tx caps, daily limits, recipient whitelist, approval thresholds. Small spends auto-execute. Large spends require human sign-off. Denied actions are blocked and logged.

Trust Layer

ERC-8004 + Delegations

On-chain agent identity gates trust decisions. MetaMask ERC-7710 caveats enforce constraints onchain. Even if the offchain layer is bypassed, onchain caveats protect the treasury. Defense-in-depth.

Execution Layer

Uniswap + x402

Policy-gated Uniswap swaps deploy yield into target tokens. x402 micropayments let other agents pay for access. MoonPay CLI provides an alternative multi-chain execution backend.

Portability

Base + Celo

Base is home (ETH staking yield). Celo proves portability (stataUSDC lending yield from Aave). Different yield source, different risk profile, identical spending constraint.

Partner stack

Every integration serves one thesis

Bounded financial authority for autonomous agents. Nothing else.

Lido wstETH

Treasury primitive

Staking yield is the source of all agent spending authority. Budget regenerates from real economic activity.

MetaMask

On-chain authority

EIP-7702 smart accounts + ERC-7710 caveats. AllowedTargets, ERC20TransferAmount, Timestamp, LimitedCalls.

Uniswap

Yield deployment

Policy-gated swaps via Trading API. Live WETH-to-USDC swap with full Permit2 EIP-712 flow on Base mainnet.

ERC-8004

Trust gate

On-chain agent identity gates every trust decision. Unverified counterparties escalate to human approval.

x402

Agent commerce

Other agents pay USDC to access treasury capabilities. The treasury becomes a discoverable service, not just a wallet.

Celo + Aave

Portability proof

stataUSDC earning lending yield. Proves the primitive works for stablecoins, not just ETH. Different asset, same boundary.

MoonPay CLI

Alt backend

Same policy gates, different execution rail. Multi-chain swaps, bridges, fiat ramps. Backend-agnostic control layer.

ENS

Identity layer

5 subdomains under morke.eth. Owner, agent, contract, deployer — all human-readable. Every API response enriched.

MCP Server

Agent interface

24 tools. Spend, trade, verify, monitor — all natively callable from Claude, Cursor, or any MCP-compatible agent.

Get started

Run it yourself

Clone, configure, and give your agent a yield-funded budget.

Humans

Deploy a treasury

git clone github.com/MorkeethHQ/delegated-agent-treasury
npm install && npm run build

# Configure
cp .env.example .env
# Set your AGENT_PRIVATE_KEY
# Set your TREASURY_ADDRESS

# Start
node --env-file=.env dist/apps/api/src/server.js

# Check treasury
curl localhost:3001/treasury
